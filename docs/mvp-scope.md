# MVP 范围确认

日期：2026-06-17  
方法：Product Design scope slice  
任务：Confirm MVP scope from product design plan

## MVP 一句话

用户上传商业空间 PNG/PDF 平面图，手动确认边界、入口、人流、柱子/后场等约束并输入 brief，系统生成 3-6 套可编辑功能落位方案，支持选择精修并导出 PNG/PDF/JSON 汇报包。

## 用户与场景

优先用户：

- 商业空间设计团队
- 室内设计工作室
- 连锁品牌门店设计团队
- 商场招商和资产运营团队
- 办公空间 test-fit 团队

MVP 核心场景：

- 设计师或品牌方拿到铺位图后，需要在方案早期快速探索多个功能落位方向。
- 用户不需要马上进入施工图深度，只需要可讨论、可编辑、可汇报的方案图。
- 用户希望历史案例能被 AI 参考，而不是每次从空白开始。

## P0 范围

P0 是 MVP 必须完成的闭环。

- 项目创建
- PNG/PDF 上传
- 图纸底图锁定
- 比例尺校准
- 手动绘制可设计区域
- 手动标记入口、人流权重、柱子、后场、不可动区域
- 项目 brief 输入
- 20-50 个种子案例准备与人工标注
- 基础案例参考，可以先用标签/规则检索
- AI 生成 3-6 个方案策略
- 策略转换为结构化 scheme JSON
- 方案卡片：预览、策略说明、评分、优缺点
- 基础 2D 编辑：移动分区、编辑文字、切换图层
- 导出 PNG、PDF、JSON
- 最终方案入库为案例

## P1 范围

P1 是提升可用性和设计工作流衔接的能力。

- SVG 导出
- DXF 导出
- RAG 相似案例检索
- 家具符号库
- 更完整的图层管理
- 对话式局部修改
- 更细的方案评分：动线、容量、展示面、成本、施工复杂度
- 案例标签批量编辑
- 方案版本对比
- 导出包 zip

## P2 范围

P2 是规模化和专业化能力。

- Windows C# Revit Add-in
- RVT 转空间 JSON
- Autodesk APS 云端 RVT 转换
- 多人协作
- 企业权限
- 私有案例库权限隔离
- 专用布局生成模型
- 自定义品牌模板

## MVP 明确不做

- 不自动生成最终施工图。
- 不承诺消防、审图、规范合规。
- 不承诺 100% 自动识别 PNG/PDF 建筑语义。
- 不直接在浏览器解析 RVT。
- 不让 LLM 直接决定不可校验的最终坐标。
- 不做完整 3D 渲染和漫游。
- 不做多人实时协作。
- 不接入会增加第三方费用的云端 RVT 转换，除非用户明确确认。
- 不优先做 C 端家装或复杂城市/建筑体量生成。

## 核心验收标准

- 用户能在 15 分钟内从图纸进入第一版方案汇报图。
- 用户能在 3 分钟内创建项目并进入图纸确认。
- 每次生成至少 3 个差异明确的方案。
- 方案元素均保存为可编辑 JSON。
- 方案不明显穿墙、挡门、压柱。
- 用户可以导出 PNG、PDF、JSON。
- 用户最终选择或修改后的方案能沉淀为案例。

## 产品原则

- AI 负责策略，几何引擎负责落位和校验。
- 人在闭环中确认边界、入口、人流和固定约束。
- 案例库是核心资产，最终方案和修改记录都应回流。
- 输出既要能汇报，也要能进入后续设计工作流。
