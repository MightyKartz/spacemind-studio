# Demo 种子案例数据集要求

日期：2026-06-17  
任务：Define demo dataset requirements: 20-50 seed cases

## 目标

准备 20-50 个高质量商业空间案例，让 AI 头脑风暴在 MVP 阶段就能参考真实布局模式，而不是只依赖通用语言模型。Phase 0 推荐目标为 32 个高质量种子案例，最低 20 个可启动，最多 50 个；宁可少而标注完整，不堆不可几何复用的漂亮图。

## 案例数量

MVP 最低：

- 20 个可用案例

推荐：

- 32 个高质量案例

每个案例都必须有人审过，不能只批量导入未校正图纸。

## 覆盖类型

业务类型建议覆盖：

| 类型 | 建议数量 | 必须覆盖 |
| --- | ---: | --- |
| 咖啡 / 茶饮 / 烘焙 | 6 | 单入口、双入口、前场社交、吧台、外摆或展示面 |
| 零售 / 服饰 / 生活方式 | 6 | 展示动线、收银、试衣/仓储、橱窗 frontage |
| 餐饮 / 轻餐 / 快餐 | 5 | 厨房/后场、排队区、取餐、客座、服务动线 |
| 办公 / 联合办公 / test-fit | 5 | 工位、会议、前台、茶水、洽谈、消防/柱网约束 |
| 展厅 / 销售中心 / 品牌展示 | 4 | 叙事动线、重点展项、洽谈、接待 |
| 快闪 / 店中店 / 柜台 | 3 | 小面积、高展示、强人流边界 |
| 混合业态 | 3 | 咖啡+零售、办公+展示、社区服务等复合功能 |

空间形态建议覆盖：

- 长条空间
- 近似矩形空间
- 双入口空间
- 单入口深进深空间
- 有柱网空间
- 有明显后场空间
- 临街入口 + 商场内入口
- 异形边界空间
- 面积段：`50-80㎡`、`80-150㎡`、`150-250㎡`、`250-500㎡`、`500㎡+`

## 每个案例必须包含

- `caseId`
- 项目类型
- 面积范围
- 空间形态
- 入口模式
- 人流权重，如果已知
- 坐标原点、比例尺、画布 crop/bounds
- 功能区列表
- zones：类型、polygon、面积、标签、颜色、优先级
- 家具 primitives：类型、bbox、rotation、容量、clearance
- 动线 arrows：起点、终点、权重、主/次通道
- 展示面、服务动线、排队区、收银/接待点
- 主动线策略
- 后场策略
- 展示面策略
- 最终方案 JSON
- 用户修改 diff
- 导出版本引用
- 缩略图
- 人工审核状态

## 标签规范

标签至少包含：

- `business:<type>`
- `shape:<type>`
- `entrance:<pattern>`
- `traffic:<pattern>`
- `zone:<zone_type>`
- `strategy:<strategy_type>`

示例：

```text
business:coffee
shape:long_rectangle
entrance:dual_opposite
traffic:street_high
zone:social_front
strategy:linear_center_aisle
```

## 质量门槛

案例进入生成库前必须满足：

- 有可设计区域边界。
- 边界闭合、无自交、方向一致，面积误差建议小于 3%。
- 坐标统一为 `mm`，比例尺误差建议小于 1%。
- 有入口位置。
- 有至少 3 个功能区。
- 功能区不明显越界。
- 固定元素有标记。
- 缩略图能说明方案。
- 最终方案 JSON 能通过 `schemas/case.schema.json`。
- 主通道默认不小于 `1200mm`，次通道默认不小于 `900mm`，允许项目级覆盖。
- 同一类型或同一形态不超过总量的 25%，避免案例库偏科。

## 冷启动建议

优先整理 20 个案例：

- 8 个咖啡/轻餐饮
- 5 个零售/展厅
- 4 个办公/test-fit
- 3 个商场铺位复合空间

整理后再扩到 50 个。
