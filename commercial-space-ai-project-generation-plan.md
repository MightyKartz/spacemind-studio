# 商业空间 AI 头脑风暴工具项目生成计划

日期：2026-06-17  
方法：Product Design 项目生成计划  
项目名称：商业空间 AI 头脑风暴工具

## 1. 产品目标

构建一个面向商业空间早期方案阶段的 AI 工具，让用户上传原始图纸或 Revit 转换后的 JSON，输入经营需求和空间约束后，系统自动生成多套可编辑功能落位方案，用于设计头脑风暴、客户沟通、招商展示和方案汇报。

一句话目标：

> 上传图纸，确认约束，AI 生成多套可编辑商业空间方案。

第一阶段只做到“AI 头脑风暴 + 可编辑方案图”，不承诺自动施工图、不承诺完整 BIM 建模。

## 2. 目标用户

优先用户：

- 商业空间设计团队
- 室内设计工作室
- 连锁品牌门店设计团队
- 商场招商与资产运营团队
- 办公空间和租赁 test-fit 团队

核心场景：

- 拿到铺位图后快速探索空间潜力。
- 方案早期需要 3-8 个不同思路。
- 需要把历史案例转化为可复用案例库。
- 需要导出带功能分区、动线、说明和图例的汇报图。

## 3. 生成链路总览

```text
用户导入图纸
  ↓
图纸解析与空间 JSON
  ↓
用户确认约束
  ↓
项目 brief 输入
  ↓
历史案例检索
  ↓
AI 生成方案策略
  ↓
几何引擎生成布局
  ↓
约束校验与方案评分
  ↓
前端渲染方案卡片
  ↓
用户选择、对话修改、手动编辑
  ↓
导出 PDF / PNG / SVG / DXF / JSON
  ↓
最终方案进入案例库
```

## 4. 输入内容

### 4.1 图纸输入

MVP 支持：

- PNG
- PDF
- SVG
- DXF
- 手动绘制边界

第二阶段支持：

- RVT 通过 Revit Add-in 转 JSON
- RVT 通过 Autodesk APS 云端转换

### 4.2 用户输入

必填：

- 项目类型：咖啡、零售、办公、餐饮、展厅、快闪等。
- 空间面积或比例尺。
- 入口位置。
- 人流权重。
- 固定区域：后场、柱子、设备、不可动墙。
- 目标偏好：座位更多、展示更强、动线更顺、社交更强、成本更低。

可选：

- 风格偏好
- 品牌调性
- 最少座位数
- 功能区清单
- 预算等级
- 参考案例
- 需要生成的方案数量

## 5. 输出内容

每次生成输出：

- 3-8 个方案卡片
- 每个方案的功能分区图
- 每个方案的策略说明
- 每个方案的评分
- 每个方案的优缺点
- 可继续编辑的方案 JSON

最终导出：

- PDF：客户汇报
- PNG：微信和轻量分享
- SVG：Figma、Illustrator、网页表达
- DXF：AutoCAD、Revit、Rhino、SketchUp 等设计工作流
- JSON：回到本系统继续编辑和沉淀案例

## 6. 核心产品模块

### 6.1 项目创建

功能：

- 创建项目
- 上传图纸
- 填写项目 brief
- 选择项目类型
- 设置单位和比例

验收标准：

- 用户可以在 3 分钟内完成项目创建。
- 上传图纸后能进入图纸确认页面。

### 6.2 图纸确认

功能：

- 显示底图
- 标记可设计区域
- 标记入口
- 标记人流方向和权重
- 标记柱子、后场、不可动区域
- 确认比例尺

验收标准：

- 用户能手动修正 AI 或系统识别错误。
- 所有约束会进入项目 JSON。

### 6.3 案例库

功能：

- 导入历史案例
- 给案例打标签
- 记录项目类型、面积、形态、入口模式、功能分区
- 保存最终方案 JSON 和缩略图

验收标准：

- 新项目能检索出相似案例。
- 案例检索结果能被 AI 用于生成策略。

### 6.4 AI 头脑风暴

功能：

- 根据项目 brief 生成多个方案方向。
- 引用相似案例的布局策略。
- 输出可解释方案。
- 支持对话式修改。

示例指令：

```text
生成 6 个适合咖啡社交空间的方案，沿街入口权重 60%，商场入口权重 40%，保留后场，增加展示面和停留感。
```

验收标准：

- 每次生成不少于 3 个方案。
- 每个方案有明确差异。
- 每个方案可继续编辑。

### 6.5 几何生成与约束校验

LLM 不直接决定最终坐标。

LLM 负责：

- 理解需求
- 生成策略
- 选择案例参考
- 解释方案

几何引擎负责：

- 分区落位
- 家具摆放
- 通道宽度
- 门洞避让
- 柱子避让
- 碰撞检测
- 边界裁剪

验收标准：

- 方案不穿墙。
- 方案不挡门。
- 方案不压柱。
- 主通道宽度符合设置。
- 家具和功能区均位于可设计区域内。

### 6.6 2D 编辑器

功能：

- 缩放和平移
- 图层开关
- 拖动功能区
- 编辑文字
- 调整家具
- 调整动线箭头
- 撤销重做
- 方案版本保存

验收标准：

- 用户能在 AI 生成基础上完成手动精修。
- 精修后的方案能保存为新的案例版本。

## 11. 验收标准

MVP 验收：

- 用户可以上传一张原始图纸。
- 用户可以标记边界、入口、人流、后场和柱子。
- 用户可以输入项目 brief。
- 系统可以生成至少 3 个方案。
- 每个方案有可视化预览和策略说明。
- 用户可以选择一个方案继续编辑。
- 用户可以导出 PNG/PDF/JSON。
- 生成方案不会明显穿墙、挡门、压柱。

产品价值验收：

- 用户能在 15 分钟内得到第一版汇报图。
- 用户愿意继续编辑 AI 生成的方案。
- 用户愿意把最终方案保存为案例。
- 用户认为方案能激发讨论，而不是只当玩具。

## 12. 风险与应对

风险：图纸识别不准  
应对：MVP 允许用户手动确认，不强依赖全自动识别。

风险：AI 方案不可落地  
应对：LLM 只生成策略，坐标由几何和约束引擎生成。

风险：案例库冷启动  
应对：先人工整理 20-50 个高质量案例作为种子库。

风险：RVT 转 JSON 难度较高  
应对：第一阶段先不阻塞 MVP，后续作为独立模块开发。

风险：用户把它期待成施工图工具  
应对：产品文案明确定位为早期方案头脑风暴和 test-fit。

## 14. 最小可行版本定义

最小可行版本不是完整 CAD，也不是完整 Revit 自动化，而是：

> 用户上传一张商业空间平面图，确认空间约束，输入需求，系统生成 3-6 套可编辑功能落位方案，并导出汇报图。

只要这个闭环成立，后续 Revit、DXF、SVG、案例库和专用模型都可以逐步叠加。
